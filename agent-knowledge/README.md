# Memoria Global del Ingeniero en Sistemas Senior

## ¿Qué es la Memoria Global?

La Memoria Global es un repositorio centralizado de conocimiento técnico, patrones, reglas y experiencias acumuladas durante el desarrollo de sistemas. Funciona como una base de conocimiento transferible que permite mantener consistencia, evitar errores recurrentes y acelerar la resolución de problemas.

## Contenido

Esta memoria contiene:

- **CORE_RULES.md**: Reglas fundamentales de desarrollo y principios operativos
- **KNOWN_FAILURES.md**: Errores conocidos y sus causas raíz documentadas
- **PATTERNS.md**: Patrones de diseño y soluciones probadas
- **RECOVERY_PLAYBOOKS.md**: Procedimientos de recuperación para situaciones críticas
- **DECISION_PRINCIPLES.md**: Principios de toma de decisiones y arquitectura

## Cómo se usa

1. **Antes de iniciar una tarea**: Revisar CORE_RULES.md y PATTERNS.md para alinear el enfoque
2. **Durante el desarrollo**: Consultar KNOWN_FAILURES.md para evitar errores conocidos
3. **Al encontrar un problema**: Verificar RECOVERY_PLAYBOOKS.md para procedimientos de recuperación
4. **Al tomar decisiones arquitectónicas**: Referenciar DECISION_PRINCIPLES.md

## Cómo se actualiza

La memoria se actualiza cuando:

- Se identifica un nuevo patrón de error o solución
- Se establece una nueva regla operativa
- Se documenta un nuevo procedimiento de recuperación
- Se toma una decisión arquitectónica significativa
- Se valida un patrón en producción

**Principio**: Documentar experiencias reales, no anticipaciones. La memoria crece con la práctica, no con la especulación.
